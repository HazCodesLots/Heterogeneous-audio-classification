# Heterogeneous Audio Classification

This repository contains models and data loading pipelines designed for heterogeneous audio classification, structured around the DCASE 2026 Task 1 (Broad Sound Taxonomy - BST). The goal is to accurately classify audio samples into 23 specific, fine-grained categories encompassing Music, Instrument Sounds, Speech, Sound Effects, and Scene Sounds.

## Data

The dataset used for this project consists of two complementary parts sourced from Freesound:

1. **BSD10k-v1.2 (Broad Sound Dataset 10k)**: A curated, expert-annotated dataset containing approximately 10,000 sounds. This provides a highly reliable, clean source of data for training and evaluation.
2. **BSD35k-CS (Broad Sound Dataset 35k - Crowd Sourced)**: A much larger but noisier collection of roughly 35,000 sounds (around 150 hours of audio). The labels here are crowd-sourced directly from Freesound uploaders, reflecting real-world labeling variability, noise, and potential inconsistencies.

### Usage in this Repository
Both dataset parts are designed to be used simultaneously. The data loading pipelines in `dataset.py` (for raw audio) and `dataset_embeddings.py` (for pre-computed features) process them as follows:
* **Concatenation**: The training scripts (e.g., `train/train.py`) accept multiple `--csv_path` and `--audio_dir` arguments. The datasets are read, processed, and merged into a single massive dataset (`pd.concat`) before training begins.
* **Confidence Filtering**: To handle the noise in the `BSD35k-CS` dataset, the loaders implement a `--confidence_threshold`. Samples with confidence scores below this threshold are automatically dropped, allowing you to filter out unreliable crowdsourced labels.
* **Unified Cross-Validation**: Once concatenated, a unified Stratified K-Fold cross-validation split is applied across the combined dataset, ensuring perfectly balanced class distributions across all folds.
These parts are loaded and evaluated together using the data loaders in `dataset.py` (for raw audio) and `dataset_embeddings.py` (for pre-computed features).

## Model Architectures

### 1. Pre-trained Foundation Models (Embeddings Extraction)
To leverage massive-scale pre-training, the repository extracts feature vectors from the following frozen foundation models:
*   **CLAP (Contrastive Language-Audio Pretraining)**: Extracts 512-dimensional representations of both the audio signal and textual metadata.
*   **PANNs (Pre-trained Audio Neural Networks)**: A robust CNN backbone that provides 2048-dimensional audio features.

### 2. SOTA Late-Fusion Classifier (Active Submission Model)
*   **AttentionFusionClassifier (3-Modality Champion: [CLAP Audio, PANNs Audio, CLAP Text])**: A custom multi-modal fusion model used to combine these embeddings into a unified predictive space.
    * **Mechanism**: It projects each different heterogeneous embedding into a shared 512-dimensional subspace and treats them as a sequence of "tokens". 
    * **Self-Attention**: It passes the sequence through a **Transformer Encoder** to allow the acoustic models to align with the semantic text metadata.
    * **Mean-Pooling Regularization**: Instead of using a `[CLS]` token (which was proven to overfit), it applies a rigid `seq.mean(dim=1)` bottleneck across the modalities before passing the result through residual MLP blocks.
    * **Ensemble**: The final CSV submission is generated by averaging the logits of 7 checkpoints trained via Stratified 5-Fold Cross Validation (`best_model-epoch=49-val_hF=0.8285.ckpt`, `best_model-epoch=82-val_hF=0.8103.ckpt`, `best_model-epoch=03-val_hF=0.7336.ckpt`, `best_model-epoch=88-val_hF=0.8378.ckpt`, `best_model-epoch=58-val_hF=0.8370.ckpt`, `best_model-epoch=71-val_hF=0.8383.ckpt`, `best_model-epoch=52-val_hF=0.8296.ckpt`).

## Evaluation Metrics

To account for the hierarchical nature of the Broad Sound Taxonomy (BST), this repository implements specific metrics (found in `metrics.py`):

*   **Hierarchical F-score (hF, hP, hR)**: The primary evaluation metrics. Instead of a binary correct/incorrect, this metric gives *partial credit* (lambda = 0.75) if the model guesses the wrong specific sub-class but gets the broad top-level parent category right. It rewards models that make "reasonable" mistakes.
*   **Second-level Accuracy (`Acc(2nd)`)**: Standard classification accuracy. The percentage of time the model guesses the exact fine-grained leaf node (out of 23 classes) perfectly.
*   **Top-level Accuracy (`Acc(top)`)**: Coarse classification accuracy. The percentage of time the model guesses the broad parent category correctly (e.g., classifying a sound as *some* type of "Music", even if the specific instrument is wrong).

## Taxonomy and Class Filtering

The dataset utilizes a hierarchical taxonomy where labels are represented by a `class_idx`. The model is designed to predict exactly 23 fine-grained leaf nodes (defined in `train/model.py`).

During data loading, specific class indices are intentionally discarded:
* **Top-Level Classes (ending in `00`)**: These represent broad parent categories (e.g., general "Music" or "Speech") that lack a specific sub-class label. They are too ambiguous for the fine-grained 23-class classification task.
* **"Other" Classes (ending in `99`)**: These represent miscellaneous sub-classes that do not fall into the main defined categories of interest. 

By filtering these out, we remove noise and ensure the model strictly learns to distinguish between the 23 target classes without being forced to learn ambiguous or out-of-distribution sounds.

## Scripts Overview

* **`dataset.py`**: The high-performance raw audio data pipeline. It loads `.wav` files, computes on-the-fly Mel-spectrograms, and applies dynamic augmentations (like random cropping and SpecAugment) for models trained from scratch.
* **`dataset_embeddings.py`**: The pre-computed embeddings data pipeline. It seamlessly fuses loaded `.npy`/`.pt` feature vectors from multiple models (CLAP, AST, etc.) to massively speed up training.
* **`evaluate.py`**: The raw-audio equivalent of `evaluate_embeddings.py`. It runs an ensemble inference loop on the evaluation dataset for architectures trained from scratch (like ConvNeXt) using raw `.wav` files.
* **`evaluate_embeddings.py`**: The inference script for the embeddings pipeline. It discovers all model checkpoints, averages their logits for a robust ensemble prediction, and generates the final CSV submission.
* **`extract_ast_embeddings.py`**: Processes raw audio to extract and save feature embeddings from a pre-trained Audio Spectrogram Transformer (AST) model.
* **`extract_clap_embeddings.py`**: Processes raw audio to extract and save audio embeddings from a pre-trained CLAP (Contrastive Language-Audio Pretraining) model.
* **`extract_panns_embeddings.py`**: Processes raw audio to extract and save audio embeddings from a pre-trained PANNs (Pre-trained Audio Neural Networks) model.
* **`extract_text_embeddings.py`**: Processes text metadata (like descriptions and tags from the CSV) through a pre-trained text encoder to generate language embeddings for multimodal fusion.
* **`extract_wavlm_embeddings.py`**: Processes raw audio to extract and save acoustic features from a pre-trained WavLM model.
* **`lightning_module.py`**: Defines the PyTorch Lightning module for the embeddings pipeline. It encapsulates the training loop, validation loop, loss calculations, and optimizer configurations.
* **`metrics.py`**: Implements the custom evaluation metrics for the DCASE challenge, including the Hierarchical F-score (hF) which gives partial credit for top-level taxonomy correctness.
* **`train_embeddings.py`**: The primary training script for the pre-computed embeddings approach. It uses the Lightning trainer to train the `AttentionFusionClassifier` on the combined, pre-computed feature vectors.
